  🔍 What the errors actually were

  1. The Turbopack error: a Next 14 codebase running on Next 16. Someone bumped
  next to ^16.2.9 and next-intl to ^4 in package.json (the npx next dev log
  showed Next 16.2.9 + Turbopack), but the code and the rest of the deps were
  still Next 14 / React 18 era. The dev log was full of:
  - params.locale errors — in Next 16, params is a Promise that must be awaited
  - middleware file convention deprecated (Next 16 renamed it to proxy)
  - a React dev-instrumentation crash (flushComponentPerformance)
  - React 18 installed where Next 16 requires React 19

  2. The "white text on white background": a hydration bug, made visible by your
  reduced-motion setting. My Reveal/Hero components branched on
  useReducedMotion() for the initial style: the server rendered sections with
  opacity:0, your client (reduced motion enabled) expected opacity:1 → hydration
  mismatch. Combined with the broken Next/React mix, hydration failed and
  sections stayed invisible — white hero text sitting on the white page.

  🔧 Fixes applied

  - Coherent Next 16 stack: React 19.2, framer-motion 12, @types 19, next lint
  script (removed in 16) → typecheck
  - Async params awaited in layout.tsx, page.tsx, generateMetadata
  - src/middleware.ts → src/proxy.ts (Next 16 convention) — deprecation gone
  - Hydration fix: initial animation state is now identical on server and
  client; reduced motion is honored via instant transitions instead → verified 0

  - Contrast hardening: hero section has a dark bg-ink base (white text never
